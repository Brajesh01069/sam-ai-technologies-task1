PIPELINE

How a face becomes data.

01

Capture

Frame is pulled from webcam, image upload, or video stream at the configured resolution.

02

Detect

TinyFaceDetector / Haar Cascade locates face regions and outputs bounding boxes with confidence.

03

Embed

A 68-point landmark mesh aligns the face; the recognition net produces a 128-D embedding vector.

04

Match

Embedding is compared against the face vault using euclidean distance to return identity + similarity %.

ABOUT BRAJESH.ORG

AI Vision, built for trust as much as accuracy.

brajesh.org AI Vision started as a research project exploring how far in-browser deep learning could go for biometric computer vision — without ever sending a face to a server. The result is a full detection-and-recognition pipeline that runs on commodity hardware, built on the same class of convolutional architectures used by FaceNet and ArcFace.

Today the platform powers access control, attendance, and visitor-recognition workflows for teams who care about both performance and privacy.

Privacy by architecture

Embeddings, not images, are what gets compared and stored.

Model transparency

Every detection ships with a confidence score — no black-box certainty.

Production-grade engineering

FastAPI services, containerized inference, documented APIs.

DEVELOPER · API REFERENCE

brajesh.org REST API

Base URL: https://api.brajesh.org/v1 — all inference endpoints accept multipart image uploads and return structured JSON.

Authentication Detect Faces Recognize Face Register User List Users Delete User Recognition Logs

POST/auth/token

Exchange admin credentials for a JWT bearer token, required on all write endpoints.

curl -X POST https://api.brajesh.org/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"••••••••"}'

→ { "access_token": "eyJhbGciOi...", "expires_in": 3600 }

POST/vision/detect

Runs the detector on an uploaded image and returns bounding boxes with confidence scores.

curl -X POST https://api.brajesh.org/v1/vision/detect \
  -H "Authorization: Bearer $TOKEN" \
  -F "image=@photo.jpg"

→ { "faces": [
      { "box": [120,84,210,210], "confidence": 0.992 }
    ], "count": 1, "latency_ms": 38 }

POST/vision/recognize

Detects faces and matches each embedding against the registered face vault.

→ { "matches": [
      { "name": "Brajesh Kumar", "similarity": 0.974, "user_id": "usr_8a2" }
    ] }

POST/users

Registers a new identity: extracts the embedding from the supplied image and stores it in the vault.

curl -X POST https://api.brajesh.org/v1/users \
  -H "Authorization: Bearer $TOKEN" \
  -F "name=Brajesh Kumar" -F "role=Admin" -F "image=@face.jpg"

GET/users

Returns all registered identities (without raw images).

DELETE/users/{id}

Removes a user and their embedding permanently from the vault.

GET/logs?limit=50

Returns recent recognition events with timestamp, matched identity, confidence and source.
